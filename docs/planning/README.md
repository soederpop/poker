# Planning Materials

This folder is for project planning materials created during collaboration, such as:

- implementation plans
- roadmap drafts
- architecture notes
- research/task breakdowns
- experiment plans
- follow-up checklists

Suggested conventions:

- one topic per file
- use descriptive filenames
- prefer dated files for iterative planning
- keep durable product docs in `docs/` top-level when they become canonical
