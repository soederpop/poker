Decision Context type

The context object passed to your strategy function every decision point. All fields are read-only.

type DecisionContext = {
  // Your hand
  heroCards: [string, string]        // ["Ah", "Kd"]
  board: string[]                    // ["Ks", "7d", "2h"]
  street: "preflop" | "flop" | "turn" | "river"
  position: "UTG" | "MP" | "CO" | "BTN" | "SB" | "BB"
  inPosition: boolean

  // Pot & betting
  toCall: number                     // chips to call (0 if checked to)
  potSize: number
  effectiveStack: number             // your remaining chips
  legalActions: PokerAction[]
  playersInHand: number
  playersLeftToAct: number

  // Board analysis (auto-computed, null on preflop)
  boardTexture: BoardTexture | null
  draws: DrawAnalysis | null

  // Action context
  facingBet: boolean
  facingRaise: boolean               // raise count >= 1
  facingThreeBet: boolean            // raise count >= 2
  raiseCount: number                 // raises this street
  checkedTo: boolean
  isAggressor: boolean               // hero made last raise
  aggressor: string | null           // playerId of last raiser
  streetHistory: Array<{ playerId: string; action: string; amount?: number }>
  spr: number                        // stack-to-pot ratio
  streetIndex: number                // 0=pre, 1=flop, 2=turn, 3=river

  // Opponent profiles (HUD-style stats from all hands at this table)
  villainProfiles: Record<string, VillainProfile>  // every player, keyed by playerId
  villainProfile: VillainProfile | null             // primary opponent

  // Equity (pre-computed)
  estimatedEquity: number            // 0–1
  potOdds: number                    // 0–1
}

StrategyDecision type

What your strategy function must return.

type StrategyDecision = {
  action: "fold" | "check" | "call" | "bet" | "raise" | "all-in"
  amount?: number       // required for bet/raise
  reason?: string       // logged for debugging
}

Minimal Example

export default function decide(context: DecisionContext): StrategyDecision {
  if (context.estimatedEquity > context.potOdds) {
    return { action: "call", reason: "pot odds" }
  }
  return { action: "fold" }
}

Hand Evaluation sync

Evaluate made hands, compare showdown strength, and normalize combo strings.

evaluateHand(cards: string[]): HandResult

Takes 5–7 cards, returns the best 5-card hand.

const result = evaluateHand(["Ah", "Kh", "Qh", "Jh", "Th"])
// result.rank     → "royal-flush"
// result.value    → numeric strength (higher = better)
// result.name     → "Royal Flush"
// result.cards    → best 5 cards used

compareHands(a: string[], b: string[]): -1 | 0 | 1

Returns 1 if hand a wins, -1 if hand b wins, 0 for a tie.

const winner = compareHands(
  ["Ah", "Kh", "Qs", "Jd", "Tc"],
  ["9s", "9d", "9h", "5c", "2d"]
)
// winner → 1 (straight beats trips)

normalizeCombo(combo: string): string

Normalizes a 2-card combo to canonical form: pairs → "AA", suited → "AKs", offsuit → "AKo".

normalizeCombo("KhAh")  // → "AKs"
normalizeCombo("9d9c")  // → "99"

Equity sync async

Calculate hand equity against ranges. Note: equity functions return 0–100, not 0–1.

equity(hand, range, board?): HandEquity sync

Quick equity estimate for a hand vs a range on a given board.

const eq = equity(["Ah", "Kh"], "AA,KK,QQ,AKs", ["Ks", "7d", "2h"])
// eq.equity   → 42.3  (percent, 0-100)
// eq.win      → 39.1
// eq.tie      → 3.2
// eq.lose     → 57.7

equityEngine async

Full Monte-Carlo engine for deeper simulations.

const result = await equityEngine.calculate({
  hand: ["Ah", "Kh"],
  range: "random",
  board: [],
  iterations: 10000
})

compareRanges(rangeA, rangeB, board?): RangeEquity async

Compare two ranges against each other. Returns equity for each side.

const result = await compareRanges(
  "AA,KK,QQ",
  "JTs,T9s,98s",
  ["Jh", "Tc", "4d"]
)
// result.equityA → 61.2  (percent)
// result.equityB → 38.8

Ranges class

Build, filter, and compare hand ranges.

Constructor

const range = new Range("AA,KK,QQ,AKs,AKo")
const range2 = new Range(["AA", "KK", "QQ"])

Properties

range.size                  // number of combos
range.percentile            // 0-1, how tight (lower = tighter)
range.strength              // 0-1, average hand strength
range.comboNames            // ["AhAs", "AhAd", ...] raw combos
range.normalizedComboNames  // ["AA", "AKs", ...] canonical

Methods

range.includes("AKs")          // → boolean
range.include("JTs")           // add combos, returns new Range
range.exclude("72o")           // remove combos, returns new Range
range.compare(otherRange)       // overlap analysis

Static Presets

Range.ultraStrong()    // AA, KK
Range.strong()         // AA-TT, AKs-AJs, AKo
Range.medium()         // ~top 20% of hands
Range.loose()          // ~top 40% of hands
Range.sklansky(1)      // Sklansky group 1-8
Range.strongestHands(10) // top N combos by strength
Range.filterCombos(fn)  // custom filter function

Example: 3-bet bluff range

const bluffRange = Range.medium()
  .exclude("AA,KK,QQ,AKs")     // remove value hands
  .include("A5s,A4s")           // add suited wheel aces

if (bluffRange.includes(normalizeCombo(context.heroCards.join("")))) {
  return { action: "raise", amount: context.potSize * 3 }
}

Analyze board texture and draw possibilities. Both are auto-populated on context but you can call them manually on any board.

type BoardTexture = {
  highCard: string           // "A", "K", etc.
  isMonotone: boolean        // all one suit
  isTwoTone: boolean         // exactly 2 suits
  isRainbow: boolean         // all different suits
  isConnected: boolean       // 3+ connected cards
  isPaired: boolean          // board has a pair
  pairOnBoard: boolean       // alias for isPaired
  isTrips: boolean           // board has trips
  flushPossible: boolean     // 3+ of one suit
  straightPossible: boolean  // 3+ connected
  suitCounts: Record<string, number>
  rankCounts: Record<string, number>
  gaps: number[]             // gaps between ranks
  maxSuitCount: number
  isHighBoard: boolean       // 2+ broadway cards
  isLowBoard: boolean        // all cards 8 or below
  isDry: boolean             // rainbow, unconnected, unpaired
}

Opponent Model type

Real-time HUD-style stats on every player at your table. The biggest edge you can get.

How it works

context.villainProfiles gives you a Record<string, VillainProfile> — stats on every player at the table, keyed by playerId.

Stats are aggregated across all hands played at this table (up to the last 50 hands loaded from disk). The more hands observed, the more reliable the reads.

context.villainProfile is a convenience shortcut pointing to the primary opponent — the current aggressor, or the first active villain if no one has bet.

VillainProfile Type

type VillainProfile = {
  // Session-wide stats (across all hands)
  vpip: number              // 0–1, voluntarily put money in pot
  pfr: number               // 0–1, preflop raise frequency
  af: number                // aggression factor (bets+raises / calls)
  aggressionFreq: number   // 0–1

  // Derived flags
  isPassive: boolean        // af < 1.0 or aggFreq < 0.25
  isAggressive: boolean     // af > 2.0 or aggFreq > 0.50
  isLoose: boolean          // vpip > 0.40
  isTight: boolean          // vpip < 0.22

  // Current hand
  calledBetCount: number
  raisedCount: number
  preflopAction: "fold" | "call" | "raise" | "unknown"

  // Confidence
  handsObserved: number     // completed hands with data
  totalActions: number
}

Practical Example

// Check if the UTG opener is a known nit before 3-betting light
const utg = context.villainProfiles["utg-player-id"]

if (utg && utg.handsObserved >= 10 && utg.isTight && utg.pfr < 0.15) {
  // UTG nit opened — they have premiums, fold our marginal hand
  return { action: "fold", reason: "UTG nit opener" }
}

// Exploit a loose-passive player by value betting thinner
const villain = context.villainProfile
if (villain && villain.isLoose && villain.isPassive) {
  // They call too much — bet for value with top pair
  return { action: "bet", amount: context.potSize * 0.75, reason: "value vs station" }
}

⚠ Confidence Note

Derived flags (isPassive, isAggressive, isLoose, isTight) require 5+ observed hands to activate. With fewer hands, all flags default to false. Always check handsObserved before making reads.

📦 Container global

The container global gives your bot access to the full Luca dependency injection container. This is the escape hatch — it lets you call out to anything you can wire up: databases, HTTP APIs, LLMs, scripts, file systems, caches, or custom services you've registered. If you can write it in TypeScript and plug it into Luca, your bot can use it to make decisions within the action clock.

What you can do with it

// Call an external API for real-time odds
const client = container.feature("httpClient")
const odds = await client.get("https://my-solver.api/equity", { hand, board })

// Query a database of solved preflop ranges
const db = container.feature("diskCache")
const gtoRange = await db.get(`ranges:${position}:${raiseCount}`)

// Ask an LLM to reason about a tricky spot
const convo = await container.conversation({ model: "gpt-4.1-mini" })
const advice = await convo.ask(`Should I call ${toCall} into ${potSize} with ${hand}?`)

// Read from the file system
const fs = container.feature("fs")
const notes = await fs.readFile("strategy-notes.md")

// Access project paths, state, utilities
container.paths     // project directory structure
container.state     // persistent state store
container.utils     // helper utilities

The only constraint is the action clock — your bot has a limited window to respond. Anything that returns in time is fair game.
